# Twse_Top100_Heat 改善條文

## 1. 目的與範圍
1. 本條文用於規範 Twse_Top100_Heat 在 365 天、24 小時連續運行條件下之穩定性、可維運性、FinMind token 成本控制與 eMMC 儲存安全。
2. 本條文適用於 crawler、backend、frontend、docker-compose 與資料目錄管理策略。
3. 與本條文衝突之一般實作慣例，應以本條文優先。

## 2. 核心目標
1. 在不破壞既有 API 合約前提下，降低 FinMind token 不必要消耗。
2. 避免 eMMC 容量與寫入放大導致壽命下降或服務中斷。
3. 建立可觀測、可降載、可回復的長期運行機制。

## 3. FinMind Token 成本控制條文
1. 買入評分批次任務必須以交易日為週期執行，不得以月週期替代。
2. 每交易日完整評分重算上限為 1 次；手動強制刷新需受冷卻時間與觸發上限限制。
3. token 不得出現在 URL query string；僅允許透過 HTTP header 傳遞。
4. 強制刷新不得先刪除舊快取；應先生成新快取檔，再以原子方式切換。
5. 若已有可用快取，API 應優先回傳現有快取，不得因背景抓取啟動回傳空集合。
6. 評分背景抓取同時僅允許一個有效工作，必須具備去重鎖。
7. 遇到上游限流時，必須採用指數退避與部分進度持久化，禁止從零重跑放大消耗。
8. 評分快取有效期應採交易日邏輯，不得設定過長造成人工強刷。
9. 候選池上限必須可配置且有硬限制，禁止無上限擴張。
10. 必須輸出每日 token 成本指標：觸發次數、成功率、覆蓋率、重試次數、平均耗時。

## 4. eMMC 365x24 容量與壽命保護條文
1. 必須設定儲存水位保護：
2. 使用率達 70% 觸發告警。
3. 使用率達 80% 啟動保護模式（降頻、禁止非必要強制刷新）。
4. 使用率達 90% 啟動緊急模式（僅保留核心讀取 API）。
5. stock_ranks 必須保留上限天數，預設 7 至 14 交易日。
6. etf_ranks 必須新增保留上限天數，預設 90 至 180 交易日。
7. buy_scores JSON 必須僅保留最近 N 天（預設 7 天），不得無限保留。
8. 每日離峰時段必須執行 SQLite 維護作業：WAL checkpoint 與可控 vacuum。
9. 不得放任 WAL 檔案無界成長，必須監控 db/wal/shm 合計大小。
10. 容器日誌必須啟用 rotation（max-size、max-file），禁止無上限成長。
11. 高頻重複 INFO 訊息應降級或採樣，避免 eMMC 寫入放大。
12. 系統重啟後不得觸發過量補寫；需有啟動保護避免 I/O 峰值。

## 5. 排程與資料一致性條文
1. 盤中股票資料更新維持每 60 秒。
2. 收盤後檢查應降頻，不得持續盤中等級輪詢。
3. 買入評分任務應在收盤後固定時刻執行，若當日資料缺失須回退最近交易日。
4. 前端 mode 切換必須對應後端 mode 查詢，不得固定單一 mode 再前端重排冒充。
5. Top 100 規格應在 API 查詢層實作，不得以 Top 360 替代。
6. updated_at 應反映資料更新時點，不得僅反映請求處理時刻。

## 6. 安全與介面條文
1. CORS 應使用環境白名單，禁止預設全開。
2. migration 錯誤不得全部吞掉；僅可忽略可預期且可驗證的重複欄位情形。
3. token 與敏感設定不得寫入日誌。
4. API contract 變更前需先更新規格文件並補齊測試。

## 7. 測試與驗收條文
1. 必須補齊排程頻率與去重測試（含交易日與補跑情境）。
2. 必須補齊強制刷新「不刪舊快取」測試。
3. 必須補齊 token header 傳遞測試（禁止 query token）。
4. 必須補齊資料清理策略測試（stock_ranks、etf_ranks、buy_scores）。
5. 必須補齊容量水位保護與降載流程測試。
6. 必須補齊 mode 對應查詢與 Top100 一致性測試。
7. 發版前需產出容量預估與 token 成本報告。

## 8. SLO 與門檻
1. 每日完整評分批次 <= 1 次（非緊急情境）。
2. 評分 API 快取命中率 >= 90%。
3. eMMC 長期穩態使用率 < 70%。
4. 任一時點必須可回退至最近一份有效評分檔。
5. 365 天運行期間不得發生因儲存耗盡造成服務中斷。

## 9. 優先級實施清單（20 項）
1. 將買入評分排程調整為每交易日 16:05。
2. 對評分任務加入每日執行上限。
3. 對手動強制刷新加入冷卻時間。
4. 將 token 由 query 改為 header。
5. 強制刷新改為原子切換，禁止先刪舊檔。
6. 評分抓取加上單工去重鎖。
7. 評分失敗改為可續傳重試，避免全量重跑。
8. 調整評分快取過期策略為交易日導向。
9. 保持評分候選池上限可配置且有硬上限。
10. 建立每日 token 成本指標紀錄。
11. 新增 etf_ranks 歷史清理機制。
12. stock_ranks 保留天數改為可配置。
13. 新增 db/wal/shm 尺寸監控與警戒。
14. 排定離峰 WAL checkpoint 作業。
15. 排定離峰 vacuum/compact 作業。
16. docker-compose 加入容器 log rotation。
17. 降低高頻重複 log 等級或採樣。
18. 修正前端 mode 對應後端查詢。
19. 將 Top 360 收斂為 Top 100。
20. 對以上條目補齊自動化測試與發版門檻。

## 10. 變更治理
1. 本條文任一項調整需附理由、風險說明與回滾方案。
2. 若與既有 ADR 衝突，需新增 ADR 說明取捨。
3. 本條文建議每月檢視一次，依實際容量與 token 成本數據修訂。
