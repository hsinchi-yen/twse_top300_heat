# 台股每日成交量與週轉率前100排名專案
## 可用資料來源與開發建議

### 1. 專案目標
本專案目標是在每天下午 4 點之後，自動取得當日台股上市與櫃買市場的成交量與週轉率資料，整理出前 100 名清單，並依據排名與成交量相對比例輸出報表。

### 2. 建議採用的資料來源

#### 2.1 FinMind
- 官方網站：https://finmind.github.io/v3/
- 特性：
  - 台股為主的開源金融資料平台。
  - 提供超過 50 種資料集，且資料每天更新。
  - 適合用 Python 快速串接與原型開發。
- 適用情境：
  - 取得日成交量、歷史價格、股本或流通資訊。
  - 計算週轉率與排名。

#### 2.2 TWSE OpenAPI
- 官方網站：https://openapi.twse.com.tw/
- 特性：
  - 臺灣證券交易所官方 OpenAPI。
  - 適合作為上市市場的官方資料來源。
  - 端點多，能取得各類交易報表與排行資料。
- 適用情境：
  - 驗證上市市場成交量排行。
  - 補強 FinMind 的結果。

#### 2.3 TPEx / 政府資料開放平台
- 政府資料開放平台相關資料集可對應到 TPEX OpenAPI。
- 特性：
  - 免費、開放、較穩定。
  - 可取得 OTC 市場的成交量與週轉率排行。
- 適用情境：
  - 櫃買市場資料取得與交叉驗證。

### 3. 資料處理建議

#### 3.1 成交量排名
- 以當日成交量作為主要排序依據。
- 取前 100 名。
- 可再加入成交量占全市場總量比例，作為輔助排序欄位。

#### 3.2 週轉率計算
- 基本公式：
  - 週轉率 = 成交量 / 流通股數 × 100
- 若資料源提供現成週轉率欄位，可直接使用。
- 若沒有，則使用成交量與股本或流通股數自行計算。

#### 3.3 排名合併
- 先分別取得成交量前 100 與週轉率前 100。
- 再合併成單一報表。
- 可加入綜合分數，讓排名更接近實務需求。

### 4. 建議技術棧
- Python 3.10+
- pandas
- requests
- FinMind SDK
- schedule 或 cron
- openpyxl（若需要 Excel）
- markdown 輸出工具

### 5. 低阻擋開發原則
- 優先使用官方 API 與開放資料。
- 避免高頻率、全市場大量網頁爬取。
- 只在收盤後執行一次排程。
- 保留重試機制與錯誤記錄。
- 若需補爬，採低頻、限量、延遲請求方式。

### 6. 推薦實作順序
1. 用 FinMind 完成資料取得與排名計算。
2. 用 TWSE OpenAPI 驗證上市市場資料。
3. 補上 TPEX / 開放資料來源。
4. 輸出 Markdown 與 CSV 報表。
5. 再加入排程、自動寄信與歷史比較。
開發方向
如果你的目標是「快、穩、免費、低風險」，我建議主線這樣做：

櫃買補強：TPEX / 政府開放資料
